Project Title: Interactive CSS Battle Game Clone

Role: You are an expert frontend developer building a React application using Vite and Tailwind CSS.

Project Objective:
Build a functional, single-page CSS Battle clone where users write HTML/CSS to replicate a target image. The app must render their code live and compare it to the target to generate a match percentage score.

Tech Stack & Libraries:

React (Vite)

Tailwind CSS (for the app's UI/layout)

@monaco-editor/react (for the code editor)

html-to-image or html2canvas (to convert the user's live output into an image)

pixelmatch (for pixel-by-pixel image comparison to calculate the score)

Data Structure (Mock Levels):
Please hardcode an array of 2 starter levels. Each level should have:

id: Number

title: String

targetHTML: A string containing the exact HTML/CSS needed to create a simple shape (e.g., a centered colored square or circle).

startingCode: A basic boilerplate string of HTML/CSS for the user to start with.

UI Layout Requirements:
Create a sleek, dark-mode split-screen interface:

Header: Show the app title, a level selector dropdown, and the user's current match score (percentage).

Left Panel (Editor): Implement the Monaco Editor where the user types their HTML/CSS.

Right Panel (Visuals): Split this vertically into two boxes.

Top Box (Target): Render the targetHTML securely or display it as an image. This is what the user is trying to build.

Bottom Box (Output): An <iframe> that dynamically updates to render the exact code the user is typing in the editor.

Core Logic & Mechanics:

Live Rendering: Bind the Monaco editor's state to the srcDoc attribute of the Output <iframe> so it updates in real-time as the user types.

Comparison Function: Create a "Submit" button. When clicked, use html-to-image to capture a screenshot of the user's Output iframe.

Scoring: Convert both the target shape and the user's captured output into canvas image data. Use pixelmatch to compare the two images. Calculate the percentage of matching pixels and display it to the user in a styled modal or alert (e.g., "You scored 98%!").

Execution Steps for Replit:
Please set up the Vite project, install the necessary dependencies (@monaco-editor/react, html-to-image, pixelmatch), and build the components step-by-step. Ensure the styling doesn't leak between the app UI and the user's iframe.

Tips for your Replit Build:
Dependency Prompts: Replit is usually great at installing dependencies automatically, but if it gets stuck, you can manually type npm install @monaco-editor/react html-to-image pixelmatch in the Replit shell.

Keep it Local First: The prompt above intentionally uses hardcoded levels rather than a database. This ensures your project works flawlessly offline or right out of the box for your presentation without worrying about backend configuration errors.

Iterate: If the AI makes a mistake with the iframe layout, simply reply to it in the chat with specific corrections (e.g., "Make the output iframe and the target box exactly 400px by 300px so the comparison works perfectly").
